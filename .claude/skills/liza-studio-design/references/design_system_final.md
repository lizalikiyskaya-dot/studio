# Дизайн-система студии Лизы Ликийской — финальная версия

Этот документ заменяет все предыдущие версии правил по типографике и дизайну. Источник истины — приложенный файл `platform_redesign_v11.html` (кликабельный макет), это документ просто объясняет логику словами.

## 1. Шрифты — три уровня

| Уровень | Шрифт | Где |
|---|---|---|
| Заголовки | **Source Serif Pro** (serif), Bold | масштхэд, H1, названия разделов, названия карточек |
| Весь остальной текст | **Inter** (sans-serif) | подписи полей, кнопки, навигация, текст для чтения (рукописи, ответы, план книги) — везде, кроме заголовков и мелких меток |
| Мелкие технические метки | **JetBrains Mono** (monospace) | статус-бейджи, номера разделов, % в таблицах, даты |

Правило: `text-transform: uppercase` запрещён везде, КРОМЕ мелких технических меток.

## 3. Палитра для пользовательского выбора цвета (трекер книг, сюжетные линии, будущее выделение текста)
Полный набор — 11 цветов, использовать именно эти значения, ничего не придумывать заново:
```
Тёмные (для точек/текста на цветной заливке):
  #4C684B — зелёный
  #1F201E — почти чёрный
  #873443 — бордовый

Светлые (для фона ячейки/тега):
  #F0E1E4 — розовый
  #E0CBE7 — лаванда
  #E2E9EE — голубой
  #E2EADC — светло-зелёный
  #E8E1DC — бежевый
  #F4DDBC — персиковый
  #F8F4F1 — светло-кремовый
  #FFFFFF — белый
```
Важно: эта палитра нужна именно для ПОЛЬЗОВАТЕЛЬСКОГО выбора (раскраска ячеек плана книги, сюжетных линий, будущее выделение текста заметками) — не использовать эти цвета для элементов самого интерфейса (кнопки, навигация), кроме случаев, явно описанных в пункте 2.

## 4. Иконки — Lucide
Используется набор Lucide (`lucide-react` для Next.js — устанавливается как обычный npm-пакет, иконки импортируются по имени, без файлов-картинок).
```
Задания       → Bookmark
Материалы     → BookOpen
Рукописи      → FileText
Упражнения    → Pencil (или PencilLine)
Принято       → Check
В процессе    → Clock
На доработке  → RotateCcw
Сворачивание сайдбара → PanelLeft
```

## 5. Расположение — во весь экран, без отступов
Сайдбар прижат к левому краю окна браузера, контент — к правому, разделены тонкой линией `var(--border)`. НЕ использовать «плавающую карточку» с отступами/скруглением/тенью вокруг всего макета — это была промежуточная версия, отклонена.

## 6. Масштхэд
«Студия художественной прозы» (Source Serif Pro, bold) → «Лизы Ликийской» (Source Serif Pro, обычное начертание, БЕЗ курсива) → тонкая длинная линия на всю ширину блока, цвет #E8E1DC (НЕ короткая бордовая линия — это была промежуточная версия).

## 7. Компоненты
- Скругления: 14px на карточках, пилюли (полное скругление) на статус-бейджах и активных кнопках
- Разделение элементов — через мягкие тени или однотонный фон, не через жирные рамки
- KPI-плашки сверху страниц (Задания и т.п.): иконка в цветном круге + крупное число + мелкий лейбл
- Сайдбар сворачивается в иконочную полоску, кнопка-тоггл — видимый квадрат с иконкой PanelLeft рядом с масштхэдом, не спрятана на краю
- Маячки непрочитанных правок — точка акцентного цвета с мягкой расходящейся пульсацией (`@keyframes`, не резкое мигание)
- Заметки — плавающая круглая кнопка (FAB) внизу справа с бейджем-счётчиком, по клику панель выезжает справа (slide-in)
