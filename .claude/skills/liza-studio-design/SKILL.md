# Дизайн-система студии Лизы Ликийской

Источник истины — файл `platform_redesign_v11.html` (кликабельный макет, если приложен к проекту) и этот документ. Перед правкой глобального CSS/токенов — после изменения сверься, что ни одна страница не осталась со старыми значениями (grep по всему репозиторию).

ВАЖНО: эта версия дизайна полностью заменяет более ранние версии (газетную/serif-heavy, промежуточную "плавающая карточка", и версию Source Serif Pro + Inter + JetBrains Mono). Если в коде встретятся шрифты Old Standard TT, PT Serif, Golos Text, Manrope, Source Serif, Inter, JetBrains Mono — это пережитки старых версий, заменить по таблице ниже. Актуальный UI kit (июль 2026): Playfair Display + Montserrat, мягкие бордовый/зелёный/графит.

## 1. Шрифты — два семейства (моноширинного БОЛЬШЕ НЕТ)
| Уровень | Шрифт | Где |
|---|---|---|
| Заголовки | **Playfair Display** (serif), Semi Bold (600) | масштхэд, H1, названия разделов, названия карточек |
| Весь остальной текст | **Montserrat** (sans-serif), Regular | подписи полей, кнопки, плашки, навигация, текст для чтения, а также мелкие метки |
| Мелкие метки (%, №, даты, статус-бейджи) | **Montserrat** с лёгким трекингом (класс `.font-mono-label` — теперь Montserrat, не mono) | номера разделов, % в трекере, даты |

Отдельного monospace-шрифта нет — kit требует Montserrat везде. Класс `.font-mono-label` сохранён (чтобы не трогать 18 файлов), но внутри уже Montserrat.
`text-transform: uppercase` запрещён везде, КРОМЕ мелких меток.

**ВАЖНО — засечки (Playfair) ТОЛЬКО для самых крупных заголовков:** масштхэд «Студия художественной прозы» (`.masthead-title`) и H1 страниц (`.page-title`). Всё остальное — Montserrat, включая: имена персонажей в карточках (досье/арки/упражнения), заголовки карточек материалов, заголовки-аккордеоны, названия секций. Класс `.heading`, `h1–h4` теперь Montserrat semibold (600), НЕ serif. Если нужен серифный заголовок — это либо `.page-title`, либо `.masthead-title`, иначе Montserrat.
«Лизы Ликийской» под масштхэдом — Montserrat, цвет золотой `--gold`, начертание 500 (НЕ serif, НЕ курсив).

## 2. Цвета интерфейса (точные значения, не выдумывать заново)
```
--bg-page (фон сайдбара):              #F8F4F1
--bg-card (фон основного контента):     #FFFFFF
--bg-surface-2 (KPI-плашки, hover):     #FAF8F4
--ink (мягкий графит, НЕ чёрный):       #444444
--ink-soft:                             #5C5B56
--ink-faint / --faded:                  #8E8B84
--gray (нейтральный серый):             #7A7979
--border / --rule:                      #E7E4DC
--accent / --wine / бордовый (primary): #8B5660   (мягкий, «пыльный»; НЕ старый #873443)
--accent-soft:                          #F0E1E4
--sage / зелёный:                       #799778   (глушёный; НЕ старый #4C684B)
--sage-soft:                            #E2EADC
--gold / золото:                        #B8944E
--gold-soft:                            #F4DDBC
```
`--accent` и `--wine` — один и тот же primary-бордовый (алиасы, оба #8B5660). Используй `--accent` в новом коде.
Активная плашка навигации, кнопка сворачивания сайдбара, линия под масштхэдом, и активный **раздел** верхнего уровня (subtab variant "page", например "Создание арки персонажа") — все используют #E8E1DC (нейтральный бежевый, НЕ бордовый заливкой).
Активный **подраздел** (subtab variant "toggle", например "Примеры"/"Рассказ-1") — залитый **золотой** #B8944E с белым текстом; неактивный подраздел — белый с золотым контуром и золотым текстом.

**ВАЖНО — проверять отдельно при каждой правке сайдбара, это уже терялось дважды:** текст пунктов меню (Задания, Материалы, Рукописи, Упражнения, Циклы рассказов и т.п.) должен быть ЧЁРНЫМ (`--ink`), не светло-серым — иначе всё выглядит бледно. Исключения: подпись «Лизы Ликийской», ссылка «← к ученикам» и заголовки групп («Мастерская книги», «Мастерская рассказов») — теперь ЗОЛОТЫЕ (`--gold`), не серые. «кабинет ученика/наставника» остаётся `--faded`.
Плашка «Режим правок» и виджет календаря — белые (#FFFFFF), не --bg-surface-2.

```
Тёмные/акцентные: #799778 (зелёный), #444444 (графит), #8B5660 (бордовый),
                  #B8944E (золото), #7A7979 (серый)
Светлые: #F0E1E4 (розовый), #E0CBE7 (лаванда), #E2E9EE (голубой), #E2EADC (светло-зелёный),
         #E8E1DC (бежевый), #F4DDBC (персиковый), #F1F1F1 (светло-серый),
         #F8F4F1 (светло-кремовый), #FFFFFF (белый)
```
Эта палитра — для пользовательского выбора (раскраска ячеек, будущие заметки/выделение текста), НЕ для элементов самого интерфейса, кроме случаев из пункта 2.

## 4. Иконки
Набор **Lucide** (`lucide-react` — обычный npm-пакет для Next.js, импорт по имени, без файлов-картинок):
```
Задания → Bookmark · Материалы → BookOpen · Рукописи → FileText · Упражнения → Pencil
Принято → Check · В процессе → Clock · На доработке → RotateCcw · Сворачивание сайдбара → PanelLeft
```

## 5. Расположение
Сайдбар прижат к левому краю окна браузера, контент — к правому, без отступов вокруг, разделены тонкой линией `var(--border)`. НЕ заворачивать всё в «плавающую карточку» со скруглением/тенью/отступами по краям экрана — это отклонённый промежуточный вариант.

## 6. Масштхэд
«Студия художественной прозы» (Playfair Display Semi Bold) → «Лизы Ликийской» (Playfair Display обычное начертание 400, без курсива) → тонкая линия на всю ширину блока цвета #E8E1DC.

## 7. Компоненты
- Скругления: 14px на карточках, полное скругление (pill) на статус-бейджах и активных кнопках. **Все кнопки — pill (`rounded-full`), не квадратные.** Это касается и «Сохранить»/«Сменить» (зелёный `--sage` pill), «принять правку» (зелёный `--sage` pill), «Войти»/«Зарегистрироваться» (бордовый `--wine` pill), Одобрить/Отклонить.
- Тумблер вкл/выкл (`ToggleSwitch`): дорожка 44×24, БЕЗ обводки; вкл — заливка `--sage` (зелёная), выкл — серый `#D8D5CE`; белый кружок 20px с мягкой тенью. НЕ маленький, НЕ с тёмно-зелёной рамкой.
- Тени мягкие и рассеянные, не жёсткие рамки
- Сайдбар: компактные строки иконка+текст (НЕ крупные залитые кнопки одна под одной), активный пункт — флэт нейтральный фон #E8E1DC, текст --ink (НЕ перекрашивается в акцентный цвет). Заголовки групп («Мастерская книги/рассказов»), ссылка «← к ученикам» и «Лизы Ликийской» — золотой `--gold`. Фон панели — `--paper` #F8F4F1 (как в эталонном макете).
- Кнопка сворачивания сайдбара в иконочную полоску — видимый квадрат с иконкой PanelLeft рядом с масштхэдом
- KPI-плашки на дашборд-страницах: иконка (Lucide, не текстовый символ) в цветном круге + крупное число + мелкий лейбл
- Маячки непрочитанных правок — точка акцентного цвета с мягкой расходящейся пульсацией (анимация, не резкое мигание)
- Заметки — плавающая круглая кнопка (FAB) в правом нижнем углу с бейджем-счётчиком, по клику панель выезжает справа

## 8. Структура меню
**Верх (крупные пункты):** Задания и обратная связь, Материалы, Рукописи, Упражнения
**Мастерская книги:** План книги, О книге, Сеттинг (+ подвкладка «Фэнтези мир» с замочком открытия для конкретного ученика), Досье персонажей, Арки персонажей (5 арок: позитивная, разочарования, падения, порчи, плоская — три последних это разные виды "отрицательной" арки по K.M. Weiland), Акты и сюжетные линии
**Мастерская рассказов** (включается/выключается отдельно для конкретного ученика, дополняет книжную, не заменяет): 01 Циклы рассказов, 02 Отдельные рассказы

## 9. Перед тем как закончить любую UI-задачу
1. Проверь страницу логина, регистрации, кабинет наставника И кабинет ученика (все вкладки) — общий компонент встречается везде
2. Прогони grep по `uppercase` и по названиям шрифтов, которых не должно быть (Old Standard TT, PT Serif, Golos Text, Manrope) — если найдёшь, замени по таблице из пункта 1
3. Прогони grep по признакам старой «плавающей карточки» (закруглённый контейнер всего макета с тенью и отступами от края экрана) — если найдёшь, убери, макет должен быть во весь экран
4. Если правишь что-то, что выглядит как разовая правка одного экрана — спроси себя, не общий ли это компонент, который повлияет на другие страницы
